# strukturapython

1.Что такое структура данных и зачем они нужны? Структура данных — это способ организации, хранения и управления данными в программе. Они нужны для эффективного доступа к данным, оптимизации использования памяти, ускорения выполнения операций (поиск, сортировка, вставка) и упрощения кода при решении сложных задач.
2.Какие операции поддерживают множества (sets) и для каких задач их лучше использовать? Множества поддерживают: add() — добавление элемента, remove()/discard() — удаление, union() — объединение, intersection() — пересечение, difference() — разность, оператор in — проверка принадлежности за O(1). Лучше использовать для: удаления дубликатов из списка, проверки уникальности, математических операций над группами данных, быстрой проверки наличия элемента.
3.Как создать пустой словарь и пустое множество? В чем особенность синтаксиса? Пустой словарь создаётся как: my_dict = {}. Пустое множество создаётся только через функцию: my_set = set(). Особенность: фигурные скобки {} без содержимого создают именно словарь, а не множество, поэтому для пустого множества обязательно использовать set().
4.Как получить все ключи, значения и пары ключ-значение из словаря? Метод keys() возвращает все ключи, values() — все значения, items() — пары (ключ, значение) в виде кортежей. Пример: for key, value in my_dict.items(): позволяет перебрать словарь попарно.
5.В чем разница между методами append() и extend() для списков? append(x) добавляет элемент x как один целый элемент в конец списка (даже если это список — он вложится как один элемент). extend(iterable) добавляет каждый элемент из итерируемого объекта по отдельности. Пример: [1,2].append([3,4]) → [1,2,[3,4]], а [1,2].extend([3,4]) → [1,2,3,4].
6.Как безопасно получить значение из словаря, если ключ может отсутствовать? Использовать метод get(key, default), который вернёт default (или None, если не указан), если ключ не найден, вместо выброса ошибки. Альтернатива: предварительно проверить наличие ключа через if key in dict.
7.Что такое списковые включения (list comprehensions) и в чем их преимущества? Это компактный синтаксис для создания списков: [выражение for элемент in итерируемый_объект if условие]. Преимущества: код короче и читаемее, выполняется быстрее обычного цикла за счёт оптимизации интерпретатора, позволяет сразу фильтровать и преобразовывать данные в одной строке.
8.Когда следует использовать кортежи вместо списков? Приведите 3 практических примера. Кортежи используют, когда данные не должны изменяться: (1) хранение координат или констант — POINT = (55.75, 37.61), (2) использование как ключей словаря — {(0,0): "центр"}, так как списки нехэшируемы, (3) возврат нескольких значений из функции — return name, age, grade (возвращается кортеж). Кортежи занимают меньше памяти и работают быстрее списков.
